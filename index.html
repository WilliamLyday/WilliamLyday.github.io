HigherLower game

Goal: Write a number guessing game, where a number is chosen between 1 and
100. The user guesses, the game indicates higher, lower, or correct. The user
continues to guess until correct. The score is always displayed as well as a top 10
high scores list. If just finished game's score was within the top 10, this
score should be displayed within that list.

Additional Requirements:
Must support SSL/TLS certificates to provide secure connection
Needs a typical login (sign up with new/existing)
Needs a db to store user login data
Needs a db to store all time high scores data
Must have crptographically hashed password with unique salting at account creation

Hangman game

Goal: Create a similar web stack application as the number guessing game, except now it's the game of hangman. The game itself should be played using an asynchronous UI A guess should not be redirected/posted/getted to a new page which generates a new next phase page Either use AJAX (client side directed asynchronous) or use websockets. Self/Free hosting are options

Additional Requirements:
Must support SSL/TLS certificates to provide secure connection
Needs a typical login (sign up with new/existing)
Needs a db to store user login data
Needs a db to store all time high scores data
Must have crptographically hashed password with unique salting at account creation
Must use a different stack; You can choose .NET or Java, but don't use a stack 3+ years old
